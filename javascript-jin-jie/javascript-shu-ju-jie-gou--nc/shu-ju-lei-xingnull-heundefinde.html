# 数据类型 null&undefined

# undefined

undefined是全局对象的一个属性。也就是说，它是全局作用域的一个变量。

全局属性undefined表示原始值undefined。它是一个JavaScript的原始数据类型。undefined类型只有一个值，就是特殊的undefined;

  • A variable that has not been assigned a value is of type undefined.
  • A method or statement also returns undefined if the variable that is being evaluated does not have an assigned value.
  • A function returns undefined if a value was not returned.

以下几种情况会出现undefined:

  1. 声明了一个变量，但未对其初始化时，这个变量的值就是undefined。
var demo1
console.log(demo);// undefined

var demo2 = undefined;
console.log(demo2 === undefined); //true

var demo3 = 1;
console.log(demo3); //1

demo3 = undefined;
console.log(demo3 === undefined); // true

不存在需要显式地把一个变量设置为undefined值的情况，因为对于未经初始化的值默认就会取得undefined值，而已经初始化的值再将其赋值为undefined来表示空值是没有意义且不可取的

  1. 对未定义的变量执行typeof操作符也会返回undefined。
var demo1;

console.log(typeof demo1); // "undefined"
console.log(typeof demo2); // "undefined"
  1. 一个函数如果没有使用return语句指定返回值，就会返回一个undefined值。
var fucDemo = (function() {
  console.log("fucDemo");
})();
console.log(fucDemo);
//fucDemo
//undefined
  1. 调用函数时没有传参数值，参数同样也会被初始化为undefined值
function test(a){
    console.log(typeof a);  // undefined
    return a;
}

test();
//undefined

if (x === undefined) {

} else {

}

# Typeof 操作符和undefined

var x;
if(typeof x === 'undefined') {
    // 执行这些语句
}

// 这里没有声明y
if(typeof y === 'undefined') {
  // 没有错误，执行结果为true
   console.log("y is " + typeof y )  
   // y is undefined
}

if(y === undefined) {
  // ReferenceError: y is not defined
}

if(x === void 0) {
    // 执行这些语句
}

// 没有声明y
if(y === void 0) {
    // 抛出一个RenferenceError错误(与`typeof`相比)
}

# null

值 null 特指对象的值未设置。它是 JavaScript 原始数据之一。null值表示一个空对象指针，指示变量未指向任何对象

# typeof null

 console.log(typeof null)
// "object"

# null判断

if(demo === null){
       console.log("demo中没有保存对象引用！");
   }

# null使用

  • 手动设置变量的值或者对象某一个属性值为null（在初始化对象时，手动设置对象为null。在作用域中不再需要使用某个对象时，把null赋值给那个变量解除引用，以释放内存
  • 、在javascript的DOM元素获取中，如果没有获取到指定的元素对象，结果一般是null。
  • Object.prototype._proto_的值也是null。
  • 、在正则捕获的时候，如果没有捕获到结果，默认也是null

# null与undefined区别

ypeof null        // "object" (因为一些以前的原因而不是'null')
typeof undefined   // "undefined"
null === undefined // false
null == undefined // true
null === null // true
null == null // true
!null //true
isNaN(1 + null) // false
isNaN(1 + undefined) // true
