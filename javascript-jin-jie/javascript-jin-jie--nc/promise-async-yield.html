# Promise-async-generator

# Generator

generator 迭代器语法中涉及的语法糖*和 yield。 *用在生成器函数的开头

在调用一个 Generator 函数后，并不会立即执行其中的代码，函数会返回一个 Generator 对象，通过调用对象的 next 函数，可以获得 yield/return 的返回值。

无论是触发了 yield 还是 return，next()函数总会返回一个带有 value 和 done 属性的对象。 value 为返回值，done 则是一个 Boolean 对象，用来标识 Generator 是否还能继续提供返回值。

  • generator 使用特殊 yield 语句来返回数据
  • yield 语句保持以前的调用状态，并从它停止的地方继续。
  • 如果 yield 在循环中使用它，它只会在每次我们在调迭代器上调用 next()方法时执行一次
  • 像 return 关键字一样，yield 关键字也会返回值
  • 但允许在 yield 之后拥有代码
  • 可以有多个 yield
  • 通过 next 方法向 generators 来回发送值
// a simple generator function
function* gen() {
  yield 1;
  yield 2;
  yield* [3, 4, 5];
}
var res = gen();
res.next(); // {value: 1, done: false}
for (var value of gen()) {
  console.log(value);
}
// result is 1,2,3,4,5

# async/await

  • async 函数返回的是一个 Promise 对象
  • async 返回一个 Promise ，所以可以通过 then 获取值
async function testAsync() {
  return "hello async";
}

const result = testAsync();
console.log(result); //Promise 对象

testAsync().then(v => {
  console.log(v); // 输出 hello async
});
  • await 后面可以是 Promise 对象或其他表达式
  • await 后面不是 Promise 对象，直接执行
  • await 后面是 Promise 对象会阻塞后面的代码，Promise 对象 resolve，然后得到 resolve 的值，作为 await 表达式的运算结果

# Promise

  • Promise 接受一个函数 handle 作为参数，handle 包括 resolve 和 reject 两个是函数的参数
  • Promise 相当于一个状态机,有三种状态:pending,fulfilled,reject,初始状态为 pending
  • 调用 resolve,状态由 pending => fulfilled
  • 调用reject，会由pending => rejected
  • then 方法可以链式调用,每次返回一个新的 Promise
  • 执行规则和错误捕获：then 的返回值如果是非 Promise 直接作为下一个新 Promise 参数，如果是 Promise 会等 Promise 执行
new Promise((resolve, reject) => {
  resolve("这是第一个 resolve 值");
})
  .then(data => {
    console.log(data); //会打印'这是第一个 resolve 值'
  })
  .catch(() => {});

# 区别

  • async 是内置执行器，Generator 函数的执行必须依靠执行器，无需手动执行next()
  • yield 命令后面只能是 Thunk 函数或 Promise 对象，而 await 后面可以是任意表达式，都会返回一个 Promise 对象
  • async 就相当于Generator 的*，await 相当于 yield，用法有很多相似之处
  • async 函数返回的是一个 Promise 对象
  • async/await 基本上是 Generators 的一个子集，并且有一个新的语法糖
  • generator 函数返回的是 Iterator 遍历器对象，所以 async 函数执行后可以继续使用 then 等方法来继续进行下面的逻辑
  • async 函数就是 Generator 函数的语法糖
